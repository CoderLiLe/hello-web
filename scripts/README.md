# 自动化脚本

本目录包含用于项目优化和管理的自动化脚本。

## 📁 脚本列表

### 1. 文档优化脚本
- **文件**: `optimize-docs.js`
- **用途**: 检查和优化Markdown文档
- **功能**:
  - 扫描所有Markdown文件
  - 检查文档格式问题
  - 添加文档头部信息
  - 运行格式化和检查工具
  - 生成优化报告

### 2. 代码优化脚本
- **文件**: `optimize-code.js`
- **用途**: 检查和优化HTML、CSS、JavaScript代码
- **功能**:
  - 扫描所有代码文件
  - 检查代码质量问题
  - 自动优化HTML文件
  - 生成代码质量报告
  - 提供优化建议

### 3. 构建脚本
- **文件**: `build.js`
- **用途**: 完整的项目构建流程
- **功能**:
  - 清理构建目录
  - 检查项目依赖
  - 构建文档网站
  - 验证构建结果
  - 生成构建报告

## 🚀 使用方法

### 通过npm脚本运行

```bash
# 优化文档
npm run optimize:docs

# 优化代码
npm run optimize:code

# 优化所有内容
npm run optimize:all

# 完整构建
npm run build:full

# 检查项目
npm run check

# 预览构建结果
npm run preview
```

### 直接运行脚本

```bash
# 文档优化
node scripts/optimize-docs.js

# 代码优化
node scripts/optimize-code.js

# 项目构建
node scripts/build.js
```

## 🔧 配置说明

### 文档优化配置
- 检查行长度: 120字符
- 自动添加文档头部
- 支持的语言标识: 所有常见编程语言
- 忽略文件: README.md, SUMMARY.md

### 代码优化配置
- HTML检查: DOCTYPE, charset, viewport, 语义化标签
- CSS检查: 选择器复杂度, 文件大小
- JavaScript检查: ES6+特性, 代码长度, 注释比例
- 忽略目录: node_modules, .git, dist, .vuepress

### 构建配置
- 构建目录: `dist/`
- 文档目录: `docs/`
- 缓存清理: VuePress缓存和临时文件
- 验证要求: index.html, 404.html, assets目录

## 🐛 故障排除

### 常见问题

1. **脚本权限问题**
   ```bash
   chmod +x scripts/*.js
   ```

2. **依赖未安装**
   ```bash
   npm install
   ```

3. **文件路径问题**
   - 确保在项目根目录运行脚本
   - 检查文件路径是否正确

4. **内存不足**
   - 对于大型项目，可能需要增加Node.js内存限制
   ```bash
   node --max-old-space-size=4096 scripts/optimize-docs.js
   ```

### 调试模式

```bash
# 显示详细输出
DEBUG=true node scripts/optimize-docs.js

# 跳过某些检查
SKIP_LINT=true node scripts/optimize-docs.js
```
